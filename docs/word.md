## 二、项目简介（295字）

轴承作为旋转机械的核心部件，其缺陷检测对保障设备安全运行至关重要。现有检测方法存在明显局限：人工目检效率低、主观性强；单一机器视觉方法受复杂光照、反光影响，对微小瑕疵易漏检；现有深度学习方法对小尺寸瑕疵识别精度不足。

本项目提出基于深度学习的工业轴承缺陷检测系统，以最新YOLOv11为基线，创新性地融合SLIC超像素分割、局部引导注意力机制（Spot-Guided Attention）、双向特征金字塔网络（BiFPN）等先进技术，攻克复杂光照、反光干扰下的微小瑕疵识别难题。通过端到端的深度学习模型，实现轴承表面缺陷（划痕、凹坑、腐蚀）的高精度、实时性识别。预期实现微小瑕疵（<1mm）检测mAP提升至0.82，系统实时处理能力≥25 FPS。研究成果符合《中国制造2025》智能维护方向，可为工业现场提供可靠的质量监控工具。

## 三、立项依据

### 3.1 研究背景与意义

轴承被称为"工业的关节"，在航空航天、轨道交通、智能制造等领域应用广泛。据统计，旋转机械故障中约45%~55%由轴承失效引起。轴承表面缺陷（如划痕、凹槽、擦伤）主要产生于生产、运输过程，直接影响装配精度与外观质量，若流入市场将严重影响设备可靠性。

目前工业现场检测手段存在明显局限：人工目检效率低、主观性强，微小瑕疵（<1mm）识别率不足60%；传统机器视觉算法（如边缘检测、阈值分割）受光照、反光、角度旋转影响显著，对低对比度、小尺寸瑕疵易漏检；现有深度学习方法在复杂工业环境下精度仍有提升空间。

应用价值分析：以某轴承生产线上每分钟检测60件轴承为例，传统人工检测需3人/班，本系统可降低至1人/班，年节省人工成本约15万元。同时，高精度检测可避免次品流入下游环节，提升产品质量，经济效益显著。

本项目提出基于深度学习的智能检测系统，聚焦于提升微小瑕疵检测精度和系统实时性，采用YOLOv11最新架构并引入多项创新改进，旨在突破现有方法的瓶颈，实现轴承表面缺陷的一体化、高精度、实时性诊断。研究成果可直接应用于轴承生产线的质量检测，对提升国产轴承可靠性、保障设备安全运行具有重要意义，符合《中国制造2025》对高端装备智能维护的发展方向。

### 3.2 国内外研究现状

（1）基于深度学习的轴承表面缺陷检测

早期方法基于传统图像处理（如边缘检测、阈值分割），但鲁棒性差。近年来，深度学习目标检测算法成为主流，如Faster R-CNN、SSD、YOLO系列等。陶欣等（2025）在《机械设计与研究》发表的研究针对YOLOv8提出融合SLIC超像素与灰度共生矩阵的纹理分析，并引入Spot-Guided注意力模块与BiFPN结构，在Bearing12数据集上实现mAP@0.5达到92.2%，有效提升了轴承瑕疵检测精度。

然而，现有研究仍存在局限：YOLOv11作为最新一代模型（Ultralytics，2025），在速度和精度上较v8有显著提升，但目前尚未见将其应用于轴承瑕疵检测的报道；复杂工业光照环境下的鲁棒性仍需提升；对小尺寸瑕疵（<1mm）的召回率有待提高。

（2）小目标检测与注意力机制研究

针对小目标检测难题，学术界提出了多种改进方案：特征金字塔网络（FPN）及其变体（PANet、BiFPN）加强多尺度特征融合；注意力机制（如SE-Net、CBAM）增强网络对关键区域的关注；数据增强技术（如MixUp、Mosaic）提升模型泛化能力。这些技术为工业缺陷检测提供了重要参考。

（3）工业质检深度学习应用现状

深度学习在工业质检领域已取得显著进展，如PCB板缺陷检测、金属表面裂纹识别、纺织品瑕疵检测等。这些研究证明了深度学习在工业场景的可行性和优越性，为轴承缺陷检测提供了技术基础。

### 3.3 本项目的创新性

本项目的创新性体现在三个维度：

**方法论创新**：首次将YOLOv11最新深度学习架构与SLIC超像素纹理分析、局部引导注意力机制深度融合，构建端到端的轴承缺陷智能检测模型，突破传统方法在复杂光照环境下精度不足的瓶颈。

**技术创新**：
• 引入纹理熵引导的区域定位策略，通过SLIC超像素分割与GLCM纹理分析，筛选高熵候选瑕疵区域，引导网络注意力聚焦；
• 设计并优化局部引导注意力（Spot-Guided Attention）模块，增强对反光、弱纹理区域的感知能力，抑制背景噪声干扰；
• 优化双向特征金字塔网络（BiFPN）结构，加强小尺度瑕疵（<1mm）的多尺度特征融合，提升微小目标检测精度；
• 采用知识蒸馏与通道剪枝技术，在保证精度的前提下将模型参数量压缩30%，实现端侧部署。

**应用创新**：构建可部署的软件检测系统，将深度学习模型转化为工业级解决方案，支持实时检测（≥25 FPS）与批量处理，具备用户友好的人机交互界面，填补轴承智能检测的工程化空白。

## 四、研究内容

### 4.1 基于改进YOLOv11的轴承表面微小瑕疵实时检测算法

• **图像预处理增强**：以YOLOv11为基线，引入超像素分割（SLIC）与灰度共生矩阵（GLCM）对图像进行预处理，计算各超像素块的纹理熵，定位高熵候选瑕疵区域，引导网络注意力聚焦；

• **注意力机制嵌入**：设计局部引导注意力（Spot-Guided Attention）模块，在特征提取阶段增强对反光、弱纹理区域的感知能力，抑制背景噪声干扰；

• **多尺度特征融合**：优化双向特征金字塔网络（BiFPN），加强小尺度瑕疵（<1mm）的多尺度特征融合，提升微小目标检测精度；

• **模型轻量化**：采用知识蒸馏与通道剪枝技术，在保证精度的前提下将模型参数量压缩30%，实现端侧部署；

• **损失函数优化**：设计自适应损失函数，缓解小目标样本类别不平衡问题，提升召回率。

### 4.2 工业环境鲁棒性增强技术

• **复杂光照处理**：研究直方图均衡化、CLAHE（对比度受限自适应直方图均衡化）等图像增强技术，提升模型在复杂光照环境下的鲁棒性；

• **抗干扰能力提升**：引入注意力机制和特征增强模块，抑制反光、阴影等干扰因素对检测结果的影响；

• **数据增强策略**：设计针对轴承缺陷的数据增强方法，包括随机旋转、亮度变换、噪声注入、MixUp等，提升模型泛化能力；

• **在线学习能力**：探索小样本学习、迁移学习技术，使模型能够快速适应新的缺陷类型和生产工况。

### 4.3 原型系统开发与工程化应用

• **软件架构设计**：基于Python + PyQt5开发用户友好的检测系统，包含图像采集、模型推理、结果显示、历史查询、数据统计等模块；

• **实时检测引擎**：采用多线程架构，图像采集线程与检测线程独立运行，实现实时检测（≥25 FPS）；

• **批量处理模式**：支持离线批量图像处理，适用于生产线质检场景；

• **可视化界面**：设计直观的用户界面，支持实时显示检测结果、缺陷标记、统计图表等；

• **系统集成测试**：在真实工业环境下进行系统测试，验证实时性、准确性和稳定性。

## 五、研究方法与技术路线

### 5.1 技术路线总体架构

本项目采用"数据采集与预处理→深度学习模型训练→性能优化→系统实现"的总体技术路线，具体如下：

1. **数据采集与预处理层**：
   • 收集公开数据集：Bearing12、NEU-DET等；
   • 自建数据集：在实验室条件下采集轴承图像，覆盖不同缺陷类型和光照条件；
   • 图像标注：使用LabelImg等工具进行人工标注，标注缺陷类别和边界框；
   • 数据增强：应用旋转、翻转、亮度变换、MixUp等增强技术扩充数据集。

2. **深度学习模型层**：
   • 基础模型搭建：基于YOLOv11构建目标检测模型；
   • 区域定位模块：SLIC超像素分割 + GLCM纹理熵计算，筛选候选区域；
   • 注意力机制：嵌入Spot-Guided Attention模块，增强关键区域特征；
   • 特征融合：优化BiFPN结构，加强多尺度特征融合；
   • 损失函数：设计自适应损失，缓解类别不平衡。

3. **模型优化层**：
   • 消融实验：验证各改进模块（注意力、BiFPN、区域定位）的有效性；
   • 超参数调优：学习率、批大小、锚框尺寸等参数优化；
   • 模型轻量化：知识蒸馏、通道剪枝，压缩模型参数量；
   • 性能测试：在不同数据集上测试模型精度和速度。

4. **系统实现层**：
   • 软件架构：Python + PyQt5 + PyTorch + OpenCV；
   • 功能模块：图像采集、实时检测、批量处理、结果统计、历史查询；
   • 实时性优化：多线程架构、GPU加速、模型量化；
   • 用户体验：直观界面、快捷操作、结果导出。

### 5.2 关键技术

1. **深度学习模型改进技术**：
   • SLIC超像素分割：将图像分割为紧凑的超像素块，计算纹理熵，定位高熵区域；
   • 局部引导注意力模块：设计空间注意力机制，动态生成权重掩膜；
   • BiFPN优化：增加同级跳跃路径连接，加强多尺度特征融合；
   • 自适应损失函数：设计针对小目标的损失权重，提升召回率。

2. **工业环境鲁棒性增强技术**：
   • 图像增强：CLAHE、直方图均衡化、Retinex算法；
   • 注意力机制：CBAM、SE-Net等注意力模块；
   • 数据增强：MixUp、CutMix、随机擦除等；
   • 对抗训练：提升模型对 adversarial examples 的鲁棒性。

3. **模型轻量化与加速技术**：
   • 知识蒸馏：以大模型为教师网络，训练小模型；
   • 通道剪枝：移除冗余通道，压缩模型参数量；
   • 量化：FP32转INT8，提升推理速度；
   • TensorRT加速：部署时使用TensorRT优化引擎。
